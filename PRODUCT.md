# Product

## Register

brand

## Users

Tech recruiters, engineering managers, and potential collaborators in the full-stack and mobile development space. They arrive via LinkedIn, GitHub, or search. Context: usually a quick scan (15-30 seconds) on desktop; decision mode on return visit. Job to be done: answer "is this person credible, skilled, and worth a conversation?"

## Product Purpose

A personal brand statement for Kathirvel S, Software Developer. Exists to convert passive awareness into active outreach. Success means a visitor remembers the name, understands the specialty (MERN Stack, React Native, Full-Stack), and feels the work is credible enough to reach out.

## Brand Personality

Technical · Creative · Driven

Voice is direct, never corporate, never playful. Confidence through specificity: real stack names, real projects, real results. No buzzword soup.

## Anti-references

- Generic Bootstrap / Tailwind portfolio template: hero photo + icon row + card grid + footer. The pattern every developer reaches for first.
- Colorful creative agency aesthetic: neon gradients, playful display type, scroll-jacked carousels, motion as decoration.
- Sterile enterprise SaaS: navy + grey palette, dashboard-flavored layout, corporate coldness that could belong to any company.

The portfolio refuses all three. Identity should be recognizable at a glance as belonging to someone specific, not to a template.

## Design Principles

1. **Specificity over decoration.** Every visual element earns its place by communicating something real about the work or the person. No filler.
2. **Quiet confidence.** Restraint signals maturity. Let the content carry weight; the layout should not compete.
3. **Engineer's precision.** Spacing, type scale, and color should feel calculated, not approximate. The interface practices what the resume preaches.
4. **Show, don't tell.** Demonstrate technical taste through implementation choices. Monospace where appropriate, exact numbers, zero rounding errors.
5. **Memorable over safe.** One distinctive choice per section beats uniform mediocrity. The portfolio should have a point of view.

## Accessibility & Inclusion

WCAG AA minimum (4.5:1 contrast normal text, 3:1 large). Respect `prefers-reduced-motion`. Keyboard navigable. Dark mode is primary; light mode must maintain the same contrast standards independently.
